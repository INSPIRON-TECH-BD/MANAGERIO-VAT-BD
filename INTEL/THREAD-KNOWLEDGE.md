## 6. PROPOSAL Q&A ANSWERS (Upwork Plain Text Optimized)

### Q1 — Business Architecture Framework (Nobin Agro Case)
**Problem:** 30 ponds, zero pond-level financial visibility, bulk feed purchases not allocated, no batch profitability, decisions on memory.
**What changed:**
- Pond-level cost centers (Projects) for batch tracking
- Production order logic: feed per pond per cycle
- Mortality accounting — dead stock as operating expense
- Lease amortization across 14,478 transactions
- Real-time executive dashboard
**Results:** 40% admin cost reduction, 100% ROI within 6 months, 0.0% error rate, business moved from memory-driven to system-driven in 6 weeks.

### Q2 — Invoice Approval in M365 + ERP (4-Layer Architecture)
1. CAPTURE (SharePoint): Document library with metadata, Power Automate trigger on upload
2. ROUTING (Power Automate): Conditional chains — <$500 dept head only, above threshold → Dept Head → Finance → CFO; reject = structured feedback loop
3. TRACKING (Planner): Each invoice = Planner task with due date, owner, status
4. ERP INTEGRATION (Sage/similar): On full approval, API push creates payable automatically — zero double-entry

### Q3 — Preventing Silos (3-Component Method)
1. Cross-department discovery before any mapping
2. Shared data objects as integration anchors (invoices, POs, projects, customer records)
3. RACI accountability map before process map
**Example:** STEP Group — mapped 3 entities as one inter-company workflow, not 3 separate departmental flows.

### Q4 — ERP Workflow Automation (X-Press Technologies)
**Automated:**
- Recurring invoices for 200+ subscribers
- Bank reconciliation via reference codes
- VAT reporting (NBR) — zero manual entry
- CPE/ONU equipment tracking at asset level
- Real-time AR aging dashboard
**Before:** Manual Excel, WhatsApp payments, manual VAT. **After:** Fully automated billing cycle, audit-ready.

### Q5 — 30–60 Day Individual→System-Driven Methodology
| Phase | Days | Actions |
|---|---|---|
| DIAGNOSTIC | 1–10 | Interview dept heads, map info flows, identify 3–5 SPOFs, document workarounds |
| ARCHITECTURE | 11–30 | Build RACI, define shared data objects, map processes dept by dept, design escalation logic |
| DEPLOYMENT | 31–60 | Configure systems, run 2-week parallel ops, role-based training, plain-language SOPs, weekly review rhythm |
**Goal by Day 60:** Business runs identically whether founder is present or not.
